## Problem Statement

Given two strings `word1` and `word2`, return the **minimum number of operations** to convert `word1` to `word2`. Allowed operations:
- **Insert** a character
- **Delete** a character
- **Replace** a character

```
Input:  word1 = "horse", word2 = "ros"
Output: 3
  horse → rorse (replace 'h' with 'r')
  rorse → rose  (delete 'r')
  rose  → ros   (delete 'e')

Input:  word1 = "intention", word2 = "execution"
Output: 5
```

## State Definition

`dp[i][j]` = minimum edits to convert `word1[0..i-1]` to `word2[0..j-1]`.

**If characters match** (`word1[i-1] == word2[j-1]`):
No operation needed — `dp[i][j] = dp[i-1][j-1]`

**If characters differ** — choose the cheapest of three operations:
- **Replace** `word1[i-1]` with `word2[j-1]`: cost = `1 + dp[i-1][j-1]`
- **Delete** `word1[i-1]` (now need to convert `word1[0..i-2]` to `word2[0..j-1]`): cost = `1 + dp[i-1][j]`
- **Insert** `word2[j-1]` into `word1` (now need to convert `word1[0..i-1]` to `word2[0..j-2]`): cost = `1 + dp[i][j-1]`

`dp[i][j] = 1 + min(dp[i-1][j-1], dp[i-1][j], dp[i][j-1])`

**Base cases**:
- `dp[i][0] = i` — convert first `i` chars of word1 to empty string: `i` deletions
- `dp[0][j] = j` — convert empty string to first `j` chars of word2: `j` insertions

## Implementation — O(m·n)

```java
public int minDistance(String word1, String word2) {
    int m = word1.length(), n = word2.length();
    int[][] dp = new int[m + 1][n + 1];

    // Base cases
    for (int i = 0; i <= m; i++) dp[i][0] = i;
    for (int j = 0; j <= n; j++) dp[0][j] = j;

    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (word1.charAt(i - 1) == word2.charAt(j - 1)) {
                dp[i][j] = dp[i-1][j-1]; // match: no cost
            } else {
                dp[i][j] = 1 + Math.min(dp[i-1][j-1],  // replace
                                Math.min(dp[i-1][j],     // delete
                                         dp[i][j-1]));   // insert
            }
        }
    }

    return dp[m][n];
}
```

`dp[5][3] = 3` ✓

## Understanding the Three Operations

At cell `(i, j)` when characters don't match:

| Operation | "From" cell | Intuition |
|---|---|---|
| Replace | `dp[i-1][j-1]` | Fix both characters simultaneously |
| Delete `word1[i]` | `dp[i-1][j]` | Remove char from word1, solve smaller subproblem |
| Insert `word2[j]` | `dp[i][j-1]` | Add char to word1 matching word2[j], solve rest |

## Complexity

| | Time | Space |
|---|---|---|
| 2D DP | O(m·n) | O(m·n) |
| Space-optimized | O(m·n) | O(n) |

Space optimization: only need the previous row. Use two 1D arrays.

## Key Insight

Edit Distance is the most general string comparison DP — it subsumes LCS. The three-operation minimum `1 + min(replace, delete, insert)` is the signature recurrence.

Note the relationship to LCS:
- LCS counts matches: edit distance penalizes mismatches
- `editDistance(s, t) = m + n - 2 * LCS(s, t)` (when only insert/delete allowed, no replace)

Applications: spell checkers (suggest words within edit distance 1 or 2), DNA sequence alignment (with biologically meaningful costs), autocorrect, fuzzy string matching.

## Edge Cases

- **One empty string**: edit distance = length of the other (all insertions or deletions).
- **Both empty**: edit distance = 0.
- **Both identical**: all cells on the diagonal are 0, returns 0.
- **Single character strings**: returns 0 (same) or 1 (replace).

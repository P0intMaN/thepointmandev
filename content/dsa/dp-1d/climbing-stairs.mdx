## Problem Statement

You are climbing a staircase with `n` steps. Each time you can climb 1 or 2 steps. In how many distinct ways can you climb to the top?

```
Input:  n = 2
Output: 2  (1+1 or 2)

Input:  n = 3
Output: 3  (1+1+1, 1+2, 2+1)
```

## The Recurrence

To reach step `n`, you came from either step `n-1` (took 1 step) or step `n-2` (took 2 steps). So:

`ways(n) = ways(n-1) + ways(n-2)`

This is exactly the Fibonacci sequence. Base cases: `ways(1) = 1`, `ways(2) = 2`.

## Approach 1: Bottom-Up DP — O(n) time, O(n) space

```java
public int climbStairs(int n) {
    if (n <= 2) return n;
    int[] dp = new int[n + 1];
    dp[1] = 1;
    dp[2] = 2;
    for (int i = 3; i <= n; i++) {
        dp[i] = dp[i-1] + dp[i-2];
    }
    return dp[n];
}
```

## Approach 2: Space-Optimized — O(n) time, O(1) space ✅

Only need the last two values:

```java
public int climbStairs(int n) {
    if (n <= 2) return n;
    int prev2 = 1, prev1 = 2;
    for (int i = 3; i <= n; i++) {
        int curr = prev1 + prev2;
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
```

## Complexity

| | Time | Space |
|---|---|---|
| Bottom-up DP | O(n) | O(n) |
| Space-optimized | O(n) | O(1) |

## Key Insight

Climbing Stairs is the entry-level DP problem because the recurrence is immediately obvious: to get here, you came from one of two prior positions. The Fibonacci structure (`f(n) = f(n-1) + f(n-2)`) underlies many staircase-variant problems:

- **#746 Min Cost Climbing Stairs** — `dp[i] = cost[i] + min(dp[i-1], dp[i-2])`
- **#509 Fibonacci Number** — same recurrence, different framing
- **#91 Decode Ways** — `dp[i] = dp[i-1] (if valid 1-digit) + dp[i-2] (if valid 2-digit)`

The generalization: if you can take `k` different step sizes, the recurrence sums `k` previous values.

## Edge Cases

- **n = 1**: return 1.
- **n = 2**: return 2.
- **Large n**: no overflow concern until n ≈ 86 (Fibonacci(87) > Integer.MAX_VALUE). For very large n, use `long`.

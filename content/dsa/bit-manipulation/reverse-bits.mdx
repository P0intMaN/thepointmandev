## Problem Statement

Reverse bits of a given 32-bit unsigned integer.

```
Input:  n = 00000010100101000001111010011100
Output:    00111001011110000010100101000000
         = 964176192

Input:  n = 11111111111111111111111111111101
Output:    10111111111111111111111111111111
         = 3221225471
```

## Approach: Bit-by-Bit Shift — O(32) = O(1)

Extract the last bit of `n` and append it to `result`:

```java
public int reverseBits(int n) {
    int result = 0;
    for (int i = 0; i < 32; i++) {
        result = (result << 1) | (n & 1); // shift result left, add last bit of n
        n >>>= 1;                          // unsigned right shift
    }
    return result;
}
```

**Each iteration:**
1. `result << 1`: make room for the new bit
2. `| (n & 1)`: put the last bit of `n` into `result`
3. `n >>>= 1`: remove the last bit from `n`

Use `>>>` (unsigned right shift), not `>>` (arithmetic), to avoid sign extension with negative numbers.

## Optimization: Divide and Conquer (for repeated calls)

If this function is called many times, cache results for 8-bit chunks:

```java
private Map<Byte, Integer> cache = new HashMap<>();

public int reverseBits(int n) {
    int result = 0;
    for (int i = 0; i < 4; i++) {
        result = (result << 8) | reverseByte((byte)(n & 0xFF));
        n >>>= 8;
    }
    return result;
}

private int reverseByte(byte b) {
    return cache.computeIfAbsent(b, k -> {
        int x = k & 0xFF; // treat as unsigned
        int rev = 0;
        for (int i = 0; i < 8; i++) {
            rev = (rev << 1) | (x & 1);
            x >>= 1;
        }
        return rev;
    });
}
```

## Key Insight

Bit reversal is mechanical — extract from the LSB and insert at the MSB of the result. The critical Java detail: use `>>>` for unsigned right shift (fills with 0) rather than `>>` (arithmetic shift, fills with sign bit).

## Edge Cases

- **n = 0**: result = 0
- **n = 1** (`00...001`): result = `10...000` (2^31 = -2147483648 as signed)
- **n = -1** (all 1s): result = -1 (all 1s reversed = all 1s)

## Problem Statement

Given a reference of a node in a **connected undirected graph**, return a **deep copy** (clone) of the graph. Each node contains a value and a list of neighbors.

```java
class Node {
    int val;
    List<Node> neighbors;
}
```

```
Input:  adjList = [[2,4],[1,3],[2,4],[1,3]]
Output: [[2,4],[1,3],[2,4],[1,3]]  (deep copy)
```

## The Challenge

You can't simply copy a node and recursively copy its neighbors — the graph can have cycles, and two nodes can share a neighbor. Without tracking what's already been cloned, you'd loop forever or create duplicate copies.

## Algorithm — DFS with HashMap

Use a `HashMap<Node, Node>` mapping original nodes to their clones. Before recursing into a neighbor, check if it's already been cloned.

```java
private Map<Node, Node> cloned = new HashMap<>();

public Node cloneGraph(Node node) {
    if (node == null) return null;

    // Already cloned — return the existing clone
    if (cloned.containsKey(node)) return cloned.get(node);

    // Create clone (without neighbors yet)
    Node clone = new Node(node.val);
    cloned.put(node, clone); // ← register BEFORE recursing (handles cycles)

    // Clone all neighbors
    for (Node neighbor : node.neighbors) {
        clone.neighbors.add(cloneGraph(neighbor));
    }

    return clone;
}
```

**Why register before recursing?** If node A and node B are neighbors of each other, cloning A will try to clone B, which tries to clone A. Without pre-registering A's clone, this would infinite loop. By registering `cloned.put(node, clone)` before recursing, the second visit to A returns the existing clone immediately.

## BFS Variant

```java
public Node cloneGraph(Node node) {
    if (node == null) return null;

    Map<Node, Node> cloned = new HashMap<>();
    Queue<Node> queue = new LinkedList<>();

    cloned.put(node, new Node(node.val));
    queue.offer(node);

    while (!queue.isEmpty()) {
        Node curr = queue.poll();
        for (Node neighbor : curr.neighbors) {
            if (!cloned.containsKey(neighbor)) {
                cloned.put(neighbor, new Node(neighbor.val));
                queue.offer(neighbor);
            }
            cloned.get(curr).neighbors.add(cloned.get(neighbor));
        }
    }

    return cloned.get(node);
}
```

## Complexity

| | Time | Space |
|---|---|---|
| DFS | O(V + E) | O(V) |
| BFS | O(V + E) | O(V) |

Every node and edge is processed exactly once.

## Key Insight

The HashMap acts as both a **visited set** (prevents cycles) and a **lookup table** (prevents duplicates). The pattern of "create clone, register, then recurse" is universal for deep-copying any structure with potential cycles — linked lists with cycles, trees with cross-pointers, etc.

## Edge Cases

- **Null input**: return null immediately.
- **Single node with no neighbors**: clone the node, neighbors list is empty.
- **Self-loop** (node is its own neighbor): handled because the node is in `cloned` before processing neighbors.
- **Multiple edges between two nodes**: each edge in `node.neighbors` is processed separately; both cloned neighbor references point to the same clone object.

## How to Spot This Pattern

Greedy works when: **making the locally optimal choice at every step leads to the globally optimal solution**. You can often verify this with an *exchange argument*: assume there's a better solution. Swap the greedy choice with the alternative. Show the result is no better — contradiction.

Signal phrases:
- *"Minimum number of intervals/meetings/platforms"*
- *"Maximum events you can attend"*
- *"Can you reach the end?"* (Jump Game)
- *"Assign tasks to minimize completion time"*
- *"Activity selection"*

## When Greedy Fails → Use DP

**Coin Change with arbitrary denominations**: greedy (take largest coin) fails for `coins=[1,3,4], amount=6` → greedy gives 4+1+1=3 coins, optimal is 3+3=2 coins. Use DP.

**0/1 Knapsack**: greedy by value/weight ratio doesn't guarantee optimal. Use DP.

**Shortest path with negative edges**: greedy (always go to cheapest neighbor) fails. Use Bellman-Ford.

The test: if a greedy choice can be "ungreedy" and improved, it's not truly greedy.

## Common Greedy Patterns

### 1. Interval Scheduling (Activity Selection)

Goal: select maximum non-overlapping intervals.
Strategy: **sort by end time**, always pick the interval that ends earliest.

```java
Arrays.sort(intervals, (a, b) -> a[1] - b[1]); // sort by end time
int count = 0, lastEnd = Integer.MIN_VALUE;
for (int[] interval : intervals) {
    if (interval[0] >= lastEnd) { // no overlap with last selected
        count++;
        lastEnd = interval[1];
    }
}
```

Why end time? Picking the earliest-finishing interval leaves the most room for future intervals.

### 2. Merge Overlapping Intervals

Strategy: **sort by start time**, merge if current start ≤ last end.

```java
Arrays.sort(intervals, (a, b) -> a[0] - b[0]);
List<int[]> merged = new ArrayList<>();
for (int[] interval : intervals) {
    if (merged.isEmpty() || interval[0] > merged.get(merged.size()-1)[1]) {
        merged.add(interval);
    } else {
        merged.get(merged.size()-1)[1] = Math.max(merged.get(merged.size()-1)[1], interval[1]);
    }
}
```

### 3. Jump Game Pattern

Track maximum reachable index. If current index exceeds it, stuck.

```java
int maxReach = 0;
for (int i = 0; i < nums.length; i++) {
    if (i > maxReach) return false; // can't reach this position
    maxReach = Math.max(maxReach, i + nums[i]);
}
return true;
```

### 4. Task Scheduling

Sort tasks/jobs by some property (deadline, processing time, ratio) to minimize total cost. The specific sort key depends on the objective.

## Proving Greedy Correctness

The **exchange argument**:
1. Assume OPT is an optimal solution that differs from the greedy solution.
2. Find the first position where they differ. OPT chose B, greedy chose A.
3. Show that swapping B for A in OPT doesn't worsen the solution.
4. Repeat until OPT matches greedy → greedy is optimal.

## Common Gotchas

- **Sort direction matters**: interval scheduling sorts by END; merge intervals sorts by START.
- **Greedy vs DP**: if the greedy choice depends on future decisions you haven't seen yet, greedy won't work.
- **Tie-breaking**: when two choices have equal greedy value, the tie-breaking rule can affect correctness.
- **Off-by-one in overlap**: `a[0] >= lastEnd` (>= means touching is OK) vs `a[0] > lastEnd` (> means touching is a new interval) — depends on whether endpoints are inclusive or exclusive.

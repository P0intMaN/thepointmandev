## Problem Statement

Given an array of intervals where `intervals[i] = [start, end]`, merge all overlapping intervals and return an array of the non-overlapping intervals that cover all the intervals in the input.

```
Input:  intervals = [[1,3],[2,6],[8,10],[15,18]]
Output: [[1,6],[8,10],[15,18]]

Input:  intervals = [[1,4],[4,5]]
Output: [[1,5]]  (touching endpoints are considered overlapping)
```

## Algorithm — Sort by Start, Merge Greedily

Sort by start time. For each interval, if it overlaps with the last interval in the result (its start ≤ last end), extend the last interval's end. Otherwise, add it as a new interval.

```java
public int[][] merge(int[][] intervals) {
    Arrays.sort(intervals, (a, b) -> a[0] - b[0]); // sort by start

    List<int[]> merged = new ArrayList<>();
    merged.add(intervals[0]);

    for (int i = 1; i < intervals.length; i++) {
        int[] last = merged.get(merged.size() - 1);
        int[] curr = intervals[i];

        if (curr[0] <= last[1]) {
            // Overlap: extend last interval's end
            last[1] = Math.max(last[1], curr[1]);
        } else {
            // No overlap: add new interval
            merged.add(curr);
        }
    }

    return merged.toArray(new int[0][]);
}
```

**Why `max(last[1], curr[1])`?** One interval could be fully contained within another (e.g., `[1,10]` and `[2,5]`). Taking `max` handles this correctly.

## Key Insight

Merge Intervals = sort by start + one-pass greedy merge. This differs from Non-overlapping Intervals (sort by end) because the goal is different: merging cares about the shape of the union, not about maximizing kept intervals.

Common follow-up — **#57 Insert Interval**: given a sorted non-overlapping list, insert a new interval and merge. Walk through the list in three phases:
1. Add all intervals that end before the new interval starts.
2. Merge all overlapping intervals with the new one.
3. Add all intervals that start after the new interval ends.

```java
public int[][] insert(int[][] intervals, int[] newInterval) {
    List<int[]> result = new ArrayList<>();
    int i = 0, n = intervals.length;

    // Phase 1: no overlap, new interval hasn't started yet
    while (i < n && intervals[i][1] < newInterval[0]) result.add(intervals[i++]);

    // Phase 2: overlapping — merge into newInterval
    while (i < n && intervals[i][0] <= newInterval[1]) {
        newInterval[0] = Math.min(newInterval[0], intervals[i][0]);
        newInterval[1] = Math.max(newInterval[1], intervals[i][1]);
        i++;
    }
    result.add(newInterval);

    // Phase 3: no overlap, new interval has ended
    while (i < n) result.add(intervals[i++]);

    return result.toArray(new int[0][]);
}
```

## Edge Cases

- **Single interval**: return as-is.
- **All overlapping**: returns one merged interval spanning min-start to max-end.
- **No overlapping**: returns the sorted original intervals.
- **Contained interval** `[1,10]` and `[2,5]`: `curr[0](2) <= last[1](10)`, extend to `max(10,5)=10` — correct, [2,5] disappears into [1,10].
- **Touching endpoints** `[1,4]` and `[4,5]`: `curr[0](4) <= last[1](4)` → merge to `[1,5]`.

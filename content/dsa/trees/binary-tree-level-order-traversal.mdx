## Problem Statement

Given the `root` of a binary tree, return the **level order traversal** of its nodes' values (i.e., from left to right, level by level).

```
Input:  root = [3,9,20,null,null,15,7]
Output: [[3],[9,20],[15,7]]

Input:  root = [1]
Output: [[1]]

Input:  root = []
Output: []
```

## Algorithm — BFS with Queue

Use a queue. Before processing each level, snapshot the current queue size — that's exactly the number of nodes at this level.

```java
public List<List<Integer>> levelOrder(TreeNode root) {
    List<List<Integer>> result = new ArrayList<>();
    if (root == null) return result;

    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);

    while (!queue.isEmpty()) {
        int levelSize = queue.size(); // snapshot: nodes at this level
        List<Integer> level = new ArrayList<>();

        for (int i = 0; i < levelSize; i++) {
            TreeNode node = queue.poll();
            level.add(node.val);

            if (node.left  != null) queue.offer(node.left);
            if (node.right != null) queue.offer(node.right);
        }

        result.add(level);
    }

    return result;
}
```

**Why `int levelSize = queue.size()` before the inner loop?** As you process level `k`, you're adding level `k+1` children into the queue. Without snapshotting `levelSize`, the queue grows mid-loop and you'd process children alongside their parents. The snapshot freezes the boundary between levels.

## Tracing the Example

`root = [3, 9, 20, null, null, 15, 7]`

```
Level 0: queue=[3]         → process 3,  add 9, 20  → [[3]]
Level 1: queue=[9,20]      → process 9,  add nothing
                           → process 20, add 15, 7  → [[3],[9,20]]
Level 2: queue=[15,7]      → process 15, add nothing
                           → process 7,  add nothing → [[3],[9,20],[15,7]]
```

## Variants Built on This Template

| Variant | Change |
|---|---|
| **#107 Level Order Bottom-Up** | Reverse `result` at the end |
| **#103 Zigzag Level Order** | Alternate adding to front/back of level list |
| **#199 Right Side View** | Take only `level.get(levelSize - 1)` per level |
| **#513 Find Bottom Left Value** | Return `level.get(0)` of the last level |
| **#111 Minimum Depth** | Return level number when first leaf is encountered |

## Complexity

| | Time | Space |
|---|---|---|
| BFS | O(n) | O(n) |

Queue holds at most O(n/2) nodes at the widest level (bottom level of a complete tree). Result list holds all `n` values.

## Key Insight

The `levelSize` snapshot is the single trick that turns a plain BFS into a level-aware BFS. Internalize this pattern — it is the answer to any problem asking about *level-by-level* or *layer-by-layer* tree processing.

## Edge Cases

- **Null root**: return empty list immediately.
- **Single node**: one iteration, `levelSize = 1`, no children added, result is `[[root.val]]`.
- **Skewed tree** (linked list): each level has one node, O(n) levels, O(1) queue size at any time.
- **Complete binary tree**: bottom level has ~n/2 nodes — that's the max queue width.

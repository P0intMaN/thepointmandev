## Problem Statement

Given the `root` of a binary tree, imagine yourself standing on the right side of it. Return the values of the nodes you can see, ordered from top to bottom.

```
Input:  root = [1,2,3,null,5,null,4]
Output: [1,3,4]

Input:  root = [1,null,3]
Output: [1,3]
```

## Approach 1: BFS — Level-by-Level, Take Last — O(n)

Standard BFS level order. The **last node at each level** is visible from the right.

```java
public List<Integer> rightSideView(TreeNode root) {
    List<Integer> result = new ArrayList<>();
    if (root == null) return result;

    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);

    while (!queue.isEmpty()) {
        int levelSize = queue.size(); // snapshot the level size

        for (int i = 0; i < levelSize; i++) {
            TreeNode node = queue.poll();

            if (i == levelSize - 1) {
                result.add(node.val); // last node in this level
            }

            if (node.left != null)  queue.offer(node.left);
            if (node.right != null) queue.offer(node.right);
        }
    }
    return result;
}
```

## Approach 2: DFS — Right-First, Record First Visit per Depth — O(n)

DFS right subtree first. The first node visited at each depth is the rightmost visible node.

```java
public List<Integer> rightSideView(TreeNode root) {
    List<Integer> result = new ArrayList<>();
    dfs(root, 0, result);
    return result;
}

private void dfs(TreeNode node, int depth, List<Integer> result) {
    if (node == null) return;

    if (depth == result.size()) {
        result.add(node.val); // first visit at this depth = rightmost
    }

    dfs(node.right, depth + 1, result); // right first
    dfs(node.left,  depth + 1, result);
}
```

**Why right-first DFS?** The first time we reach a new depth, it's always the rightmost node visible from the right side.

## Left Side View

For the left side view, just change:
- BFS: take the **first** node at each level (i == 0)
- DFS: traverse **left first**, record first visit per depth

## Complexity

| | Time | Space |
|---|---|---|
| BFS | O(n) | O(w) — width of tree |
| DFS | O(n) | O(h) — height of tree |

For a balanced tree: width ≈ n/2, height ≈ log n → DFS uses less space. For a skewed tree: width = 1, height = n → BFS uses less space.

## Key Insight

The right side view is just "the last node at each level." BFS makes this trivial with the level-snapshot pattern. The DFS approach is more elegant: it exploits the fact that right-first DFS visits nodes in the exact order they'd appear in the right side view.

## Edge Cases

- **Empty tree**: return `[]`
- **Only left subtree**: left subtree nodes are visible at deeper levels
- **Only right subtree**: all nodes visible
- **Perfect binary tree**: one node per level from the right path

## Problem Statement

Given an integer array `nums`, return `true` if any value appears **at least twice**, and `false` if every element is distinct.

```
Input:  nums = [1, 2, 3, 1]
Output: true

Input:  nums = [1, 2, 3, 4]
Output: false

Input:  nums = [1, 1, 1, 3, 3, 4, 3, 2, 4, 2]
Output: true
```

## Approaches

### 1. Brute Force — O(n²)

Check every pair `(i, j)` where `i ≠ j`.

```java
public boolean containsDuplicate(int[] nums) {
    for (int i = 0; i < nums.length; i++) {
        for (int j = i + 1; j < nums.length; j++) {
            if (nums[i] == nums[j]) return true;
        }
    }
    return false;
}
```

Works but doesn't scale — 10,000 elements means 50 million comparisons.

---

### 2. Sort First — O(n log n) time, O(1) space

Duplicates must be adjacent in a sorted array. One pass is enough after sorting.

```java
public boolean containsDuplicate(int[] nums) {
    Arrays.sort(nums);
    for (int i = 0; i < nums.length - 1; i++) {
        if (nums[i] == nums[i + 1]) return true;
    }
    return false;
}
```

Better time complexity, but modifies the original array. Acceptable when O(1) extra space is required.

---

### 3. HashSet — O(n) time, O(n) space ✅ Optimal

Add each number to a set. If it's already there, we found our duplicate.

```java
public boolean containsDuplicate(int[] nums) {
    Set<Integer> seen = new HashSet<>();
    for (int num : nums) {
        if (!seen.add(num)) return true; // add() returns false if element already present
    }
    return false;
}
```

`Set.add()` returns `false` when the element already exists — no need for a separate `contains()` call, which would otherwise cost two lookups per iteration.

## Complexity

|               | Time       | Space |
|---------------|------------|-------|
| Brute Force   | O(n²)      | O(1)  |
| Sort          | O(n log n) | O(1)  |
| HashSet       | O(n)       | O(n)  |

## Key Insight

This is the simplest demonstration of the **frequency detection** signal. Any time you see:

> "Does X appear more than once?"
> "Have I seen this before?"
> "Are there any duplicates?"

your instinct should be `HashSet`. One pass, O(1) lookup, done.

## Edge Cases

- **Single element array** `[5]` → always `false` (nothing to compare against).
- **All same elements** `[3, 3, 3]` → returns `true` on the second element.
- **Negative numbers** → `HashSet<Integer>` handles negatives without any special treatment.

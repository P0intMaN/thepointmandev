## Approach: Find Middle + Reverse + Merge â€” O(n)

Three steps:
1. **Find middle**: slow/fast pointers
2. **Reverse second half**: in-place reversal
3. **Merge two halves**: interleave

```java
public void reorderList(ListNode head) {
    if (head == null || head.next == null) return;

    // Step 1: Find middle (end of first half)
    ListNode slow = head, fast = head;
    while (fast.next != null && fast.next.next != null) {
        slow = slow.next;
        fast = fast.next.next;
    }
    // slow is now the end of the first half

    // Step 2: Reverse second half
    ListNode secondHalf = slow.next;
    slow.next = null; // disconnect first and second half
    secondHalf = reverse(secondHalf);

    // Step 3: Merge
    ListNode first = head, second = secondHalf;
    while (second != null) {
        ListNode tmp1 = first.next;
        ListNode tmp2 = second.next;
        first.next = second;
        second.next = tmp1;
        first = tmp1;
        second = tmp2;
    }
}

private ListNode reverse(ListNode head) {
    ListNode prev = null, curr = head;
    while (curr != null) {
        ListNode next = curr.next;
        curr.next = prev;
        prev = curr;
        curr = next;
    }
    return prev;
}
```

## Why Disconnect the Two Halves?

Without disconnecting (`slow.next = null`), the merged list would still have a stale pointer from the end of the first half into the original second half, creating a cycle or incorrect structure.

## Key Insight

Reorder List combines three fundamental linked list operations: slow/fast pointer (find middle), reversal, and merging. None of these requires extra space. The key is recognizing that the desired order is first-half interleaved with reversed second-half.

## Problem Statement

Given an array of intervals where `intervals[i] = [start_i, end_i]`, return the minimum number of intervals you need to remove to make the rest non-overlapping.

```
Input:  intervals = [[1,2],[2,3],[3,4],[1,3]]
Output: 1  (remove [1,3])

Input:  intervals = [[1,2],[1,2],[1,2]]
Output: 2  (keep one, remove two)

Input:  intervals = [[1,2],[2,3]]
Output: 0  (touching at 2 is NOT overlapping)
```

## Approach: Greedy — Sort by End Time — O(n log n)

This is the classic **Activity Selection Problem**. To keep the maximum number of non-overlapping intervals (= minimize removals), greedily pick the interval that **ends earliest**.

```java
public int eraseOverlapIntervals(int[][] intervals) {
    Arrays.sort(intervals, (a, b) -> a[1] - b[1]); // sort by end time

    int kept = 1; // keep the first interval
    int lastEnd = intervals[0][1];

    for (int i = 1; i < intervals.length; i++) {
        if (intervals[i][0] >= lastEnd) {
            // No overlap: keep this interval
            kept++;
            lastEnd = intervals[i][1];
        }
        // else: overlap — skip (remove) this interval
    }

    return intervals.length - kept;
}
```

**Why sort by end?** An interval ending earlier leaves more room for future intervals. Keeping the earliest-ending interval is always at least as good as keeping any other overlapping interval — the greedy exchange argument confirms this.

## Overlap Condition

`intervals[i][0] >= lastEnd` means no overlap (strict ≥). If `start == lastEnd`, the intervals merely touch → not overlapping → keep both.

## Connection to LIS

Minimum intervals to remove = n - (maximum non-overlapping intervals = LIS length on sorted end times). The greedy approach gives the LIS length in O(n) after sorting.

## Complexity

| | Time | Space |
|---|---|---|
| **Greedy (sort by end)** | **O(n log n)** | **O(1)** |
| DP | O(n²) | O(n) |

## Key Insight

"Minimum removals for non-overlapping" = n - "maximum non-overlapping intervals". To maximize the number kept, use the activity selection greedy: always take the interval with the earliest end time among those not overlapping the last kept interval.

## Edge Cases

- **No overlaps**: return 0
- **All identical intervals** `[1,2],[1,2]`: sort by end → same end → keep 1, remove n-1
- **Touching intervals** `[1,2],[2,3]`: `2 >= 2` → non-overlapping, both kept
- **Completely contained** `[1,10],[2,3]`: after sorting by end, `[2,3]` comes first → keep it, then `[1,10]` overlaps → remove `[1,10]`. Better choice!

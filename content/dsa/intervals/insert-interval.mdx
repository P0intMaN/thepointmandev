## Problem Statement

You are given an array of non-overlapping intervals `intervals` sorted by start time, and a `newInterval`. Insert `newInterval` and merge any overlapping intervals.

```
Input:  intervals = [[1,3],[6,9]], newInterval = [2,5]
Output: [[1,5],[6,9]]

Input:  intervals = [[1,2],[3,5],[6,7],[8,10],[12,16]], newInterval = [4,8]
Output: [[1,2],[3,10],[12,16]]
```

## Approach: Three-Phase Scan — O(n)

Process intervals in three phases:
1. **Before the new interval**: add all intervals that end before `newInterval.start`
2. **Overlapping**: merge all intervals that overlap with `newInterval`
3. **After the new interval**: add all remaining intervals

```java
public int[][] insert(int[][] intervals, int[] newInterval) {
    List<int[]> result = new ArrayList<>();
    int i = 0, n = intervals.length;

    // Phase 1: add all intervals ending before newInterval starts
    while (i < n && intervals[i][1] < newInterval[0]) {
        result.add(intervals[i++]);
    }

    // Phase 2: merge all overlapping intervals
    while (i < n && intervals[i][0] <= newInterval[1]) {
        newInterval[0] = Math.min(newInterval[0], intervals[i][0]);
        newInterval[1] = Math.max(newInterval[1], intervals[i][1]);
        i++;
    }
    result.add(newInterval); // add the merged interval

    // Phase 3: add all remaining intervals
    while (i < n) {
        result.add(intervals[i++]);
    }

    return result.toArray(new int[result.size()][]);
}
```

**Overlap condition**: `intervals[i][0] <= newInterval[1]` — current interval starts at or before new interval ends. Combined with Phase 1's exit condition (current interval ends at or after new interval starts), this defines overlap.

## Walkthrough: `[[1,2],[3,5],[6,7],[8,10],[12,16]]`, newInterval `[4,8]`

- Phase 1: [1,2] ends at 2 < 4 → add. [3,5] ends at 5 ≥ 4, stop.
- Phase 2: [3,5] starts 3 ≤ 8 → merge. newInterval = [min(4,3), max(8,5)] = [3,8]. [6,7] starts 6 ≤ 8 → merge. newInterval = [3,8]. [8,10] starts 8 ≤ 8 → merge. newInterval = [3,10]. [12,16] starts 12 > 8, stop.
- Add [3,10].
- Phase 3: add [12,16].
- Result: [[1,2],[3,10],[12,16]] ✓

## Complexity

| | Time | Space |
|---|---|---|
| **Three-phase scan** | **O(n)** | **O(n)** |
| Binary search + merge | O(n) | O(n) |

## Key Insight

Since the input is already sorted, no sorting step is needed. The three-phase scan processes each interval exactly once. Phase 2's merge loop simply extends the new interval's boundaries as it absorbs overlapping intervals.

The overlap condition in Phase 2 (`intervals[i][0] <= newInterval[1]`) and the exit in Phase 1 (`intervals[i][1] < newInterval[0]`) together precisely define the overlapping region.

## Edge Cases

- **No overlaps**: new interval is added as-is between two non-adjacent intervals
- **Empty intervals**: return `[[newInterval]]`
- **newInterval contains all existing**: one merged interval returned
- **Touching endpoints** `[1,3]` + `[3,5]`: `intervals[i][0]=3 <= newInterval[1]=3` → merged into `[1,5]`

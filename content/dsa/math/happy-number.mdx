## Problem Statement

A **happy number** is defined by the following process: starting with any positive integer, replace the number by the sum of the squares of its digits. Repeat until the number either equals 1 (happy) or loops endlessly in a cycle that does not include 1 (unhappy). Return `true` if the number is happy.

```
Input:  n = 19
Output: true
Explanation: 1² + 9² = 82 → 8²+2² = 68 → 6²+8² = 100 → 1²+0²+0² = 1 ✓

Input:  n = 2
Output: false
```

## Approach 1: Floyd's Cycle Detection — O(log n), O(1) space

Unhappy numbers eventually cycle. Use fast/slow pointers on the sequence:

```java
public boolean isHappy(int n) {
    int slow = n, fast = sumSquares(n);
    while (fast != 1 && fast != slow) {
        slow = sumSquares(slow);
        fast = sumSquares(sumSquares(fast));
    }
    return fast == 1;
}

private int sumSquares(int n) {
    int sum = 0;
    while (n > 0) {
        int digit = n % 10;
        sum += digit * digit;
        n /= 10;
    }
    return sum;
}
```

If `fast == 1`: happy number. If `fast == slow` (cycle detected): unhappy.

## Approach 2: HashSet — O(log n), O(log n) space

```java
public boolean isHappy(int n) {
    Set<Integer> seen = new HashSet<>();
    while (n != 1 && !seen.contains(n)) {
        seen.add(n);
        n = sumSquares(n);
    }
    return n == 1;
}
```

## Sum of Squares Computation

For a number `n`, computing `sumSquares` takes O(log₁₀ n) time (one step per digit). The sequence shrinks rapidly — a 3-digit number's sum of squares is at most `9²+9²+9² = 243`, so after the first step, values are bounded by ~500.

## Key Insight

The "sum of squares of digits" operation maps all large numbers into a small bounded range. In this range, either the sequence reaches 1 or enters the known cycle `4 → 16 → 37 → 58 → 89 → 145 → 42 → 20 → 4`. Floyd's cycle detection avoids storing all visited numbers.

## Edge Cases

- **n = 1**: immediately happy
- **n = 7**: 7→49→97→130→10→1 — happy
- **All single-digit numbers**: {1,7} are happy, the rest are unhappy

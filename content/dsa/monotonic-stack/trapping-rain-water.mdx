## Problem Statement

Given `n` non-negative integers representing an elevation map where the width of each bar is 1, compute how much water it can trap after raining.

```
Input:  height = [0,1,0,2,1,0,1,3,2,1,2,1]
Output: 6

Input:  height = [4,2,0,3,2,5]
Output: 9
```

## Approach 1: Two Pointers — O(n) time, O(1) space

Water at position `i` = `min(maxLeft[i], maxRight[i]) - height[i]`. Instead of precomputing both arrays, use two pointers — process whichever side has the smaller max.

```java
public int trap(int[] height) {
    int left = 0, right = height.length - 1;
    int maxLeft = 0, maxRight = 0;
    int water = 0;

    while (left < right) {
        if (height[left] <= height[right]) {
            // Left side is the bottleneck
            if (height[left] >= maxLeft) {
                maxLeft = height[left]; // new max, no water here
            } else {
                water += maxLeft - height[left]; // trapped water
            }
            left++;
        } else {
            // Right side is the bottleneck
            if (height[right] >= maxRight) {
                maxRight = height[right];
            } else {
                water += maxRight - height[right];
            }
            right--;
        }
    }
    return water;
}
```

**Why this works**: when `height[left] <= height[right]`, the right wall is guaranteed to be at least as tall as any wall we'll encounter from the left. So `maxLeft` alone determines how much water is trapped at `left`. The right pointer might move and find a taller wall later, but it can only increase capacity — the left side is the binding constraint.

## Approach 2: Monotonic Stack — O(n) time, O(n) space

Process water horizontally layer by layer. When we find a bar taller than the stack top, it forms the right wall of a "container". The previous element is the floor, and the current stack top (after popping) is the left wall.

```java
public int trap(int[] height) {
    Deque<Integer> stack = new ArrayDeque<>();
    int water = 0;

    for (int i = 0; i < height.length; i++) {
        while (!stack.isEmpty() && height[i] > height[stack.peek()]) {
            int floor = stack.pop(); // the low point (bottom of the container)
            if (stack.isEmpty()) break; // no left wall

            int left = stack.peek();
            int width = i - left - 1;
            int boundedHeight = Math.min(height[left], height[i]) - height[floor];
            water += width * boundedHeight;
        }
        stack.push(i);
    }
    return water;
}
```

**Visualization** for `[4,2,0,3,2,5]`:
- When we reach index 3 (height=3), pop index 2 (height=0):
  - floor=0, left=1 (height=2), right=3 (height=3)
  - width=1, boundedHeight = min(2,3)-0 = 2, water += 2
- Then pop index 1 (height=2):
  - floor=2, left=0 (height=4), right=3 (height=3)
  - width=2, boundedHeight = min(4,3)-2 = 1, water += 2

## Approach 3: Prefix Arrays — O(n) time, O(n) space

The intuitive solution: precompute `maxLeft[i]` and `maxRight[i]` for every index.

```java
public int trap(int[] height) {
    int n = height.length;
    int[] maxLeft = new int[n], maxRight = new int[n];

    for (int i = 1; i < n; i++)
        maxLeft[i] = Math.max(maxLeft[i-1], height[i-1]);
    for (int i = n-2; i >= 0; i--)
        maxRight[i] = Math.max(maxRight[i+1], height[i+1]);

    int water = 0;
    for (int i = 0; i < n; i++)
        water += Math.max(0, Math.min(maxLeft[i], maxRight[i]) - height[i]);
    return water;
}
```

## Complexity

| Approach | Time | Space |
|---|---|---|
| Prefix arrays | O(n) | O(n) |
| Monotonic stack | O(n) | O(n) |
| **Two pointers** | **O(n)** | **O(1)** ← optimal |

## Key Insight

Water at each position is bounded by the minimum of its left and right walls. The two-pointer approach avoids storing both wall arrays by reasoning: whichever side has the smaller max is the binding constraint, regardless of what the other pointer will find.

The monotonic stack fills water **horizontally** (layer by layer by container), while the prefix approach fills water **vertically** (column by column).

## Edge Cases

- **Single bar or two bars**: no container can form → 0 water
- **Monotonically increasing/decreasing**: no water trapped → 0
- **Flat terrain**: `height[i] = 0` everywhere → 0
- **One tall bar surrounded by short**: trapped water fills up to the shorter surrounding bars

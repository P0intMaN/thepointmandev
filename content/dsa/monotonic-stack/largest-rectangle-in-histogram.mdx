## Problem Statement

Given an array of integers `heights` representing the heights of bars in a histogram (each bar width = 1), find the area of the largest rectangle.

```
Input:  heights = [2,1,5,6,2,3]
Output: 10  (the rectangle of height 5 spanning bars 2–3)

Input:  heights = [2,4]
Output: 4
```

## Approach: Monotonic Increasing Stack — O(n)

For each bar, we want to know how far left and right it can extend at its height. A bar is limited by the nearest **shorter** bar on each side.

Use a monotonically increasing stack. When we encounter a bar shorter than the top, pop — the popped bar can't extend further right. The current index is its right boundary; the new stack top is its left boundary.

```java
public int largestRectangleArea(int[] heights) {
    int n = heights.length;
    Deque<Integer> stack = new ArrayDeque<>(); // increasing stack of indices
    int maxArea = 0;

    for (int i = 0; i <= n; i++) {
        // Sentinel: height 0 at end forces all remaining bars to be popped
        int currHeight = (i == n) ? 0 : heights[i];

        while (!stack.isEmpty() && currHeight < heights[stack.peek()]) {
            int height = heights[stack.pop()];
            // Left boundary: new stack top (or -1 if empty)
            int width = stack.isEmpty() ? i : i - stack.peek() - 1;
            maxArea = Math.max(maxArea, height * width);
        }
        stack.push(i);
    }
    return maxArea;
}
```

**Walkthrough** `[2,1,5,6,2,3]`:
- i=0: push 0. Stack: [0]
- i=1 (1 < 2): pop 0, height=2, width=1 (stack empty, width=1), area=2. Push 1. Stack: [1]
- i=2: push 2. Stack: [1,2]
- i=3: push 3. Stack: [1,2,3]
- i=4 (2 < 6): pop 3, height=6, width=4-2-1=1, area=6. (2 < 5): pop 2, height=5, width=4-1-1=2, area=10. Push 4. Stack: [1,4]
- i=5: push 5. Stack: [1,4,5]
- i=6 (sentinel 0): pop 5, height=3, width=6-4-1=1, area=3. Pop 4, height=2, width=6-1-1=4, area=8. Pop 1, height=1, width=6 (stack empty), area=6.
- maxArea = **10**

## Width Calculation

When popping index `j` at position `i`:
- **Right boundary**: `i` (current position that caused the pop)
- **Left boundary**: `stack.peek()` (the index just below in the stack, which is the nearest shorter bar to the left)
- **Width**: `i - stack.peek() - 1`
- If stack is empty: the bar extends to the left wall → width = `i`

## Complexity

| Approach | Time | Space |
|---|---|---|
| Brute force (check all pairs) | O(n²) | O(1) |
| **Monotonic stack** | **O(n)** | **O(n)** |

## Key Insight

A bar's maximum rectangle is determined by the nearest shorter bar on each side. The monotonic increasing stack maintains exactly this invariant — each pop event reveals both boundaries of the popped bar's maximum rectangle.

The sentinel `0` at position `n` is critical: it forces all remaining bars to be processed at the end without needing a separate loop.

## Edge Cases

- **Uniform height** `[3,3,3]`: sentinel pops all at the end. The last bar has width=3, giving area=9.
- **Strictly increasing** `[1,2,3,4,5]`: no pops until sentinel. Area = max rectangle is n*min or middle bar wide rectangle.
- **Single bar**: area = height[0].
- **All same height**: area = n * height.

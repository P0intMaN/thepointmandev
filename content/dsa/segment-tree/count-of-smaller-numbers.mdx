## Problem Statement

Given an integer array `nums`, return an integer array `counts` where `counts[i]` is the number of smaller elements to the **right** of `nums[i]`.

```
Input:  nums = [5,2,6,1]
Output: [2,1,1,0]
Explanation:
  5: [2,6,1] → 2 elements smaller (2 and 1)
  2: [6,1]   → 1 element smaller (1)
  6: [1]     → 1 element smaller (1)
  1: []      → 0 elements smaller
```

## Approach: BIT with Coordinate Compression — O(n log n)

Process elements **right to left**. For each element, query the BIT for how many elements already inserted are smaller. Then insert the element.

Since values can be large, **coordinate compress** them first: map values to ranks `[1, n]`.

```java
public List<Integer> countSmaller(int[] nums) {
    int n = nums.length;

    // Coordinate compression: rank values 1..n
    int[] sorted = nums.clone();
    Arrays.sort(sorted);
    // sorted is deduplicated by rank (use sorted index + 1 as rank)
    Map<Integer, Integer> rank = new HashMap<>();
    int r = 1;
    for (int val : sorted) rank.putIfAbsent(val, r++);
    int maxRank = r - 1;

    int[] bit = new int[maxRank + 1];
    Integer[] result = new Integer[n];

    for (int i = n - 1; i >= 0; i--) {
        int rk = rank.get(nums[i]);
        result[i] = query(bit, rk - 1); // count elements with rank < rk
        update(bit, rk, maxRank);        // insert this element
    }

    return Arrays.asList(result);
}

private void update(int[] bit, int i, int n) {
    for (; i <= n; i += i & (-i)) bit[i]++;
}

private int query(int[] bit, int i) {
    int sum = 0;
    for (; i > 0; i -= i & (-i)) sum += bit[i];
    return sum;
}
```

**Coordinate compression** maps values like `[-100, 5000000]` to ranks `[1, 2]`, making the BIT index range tight.

## Alternative: Merge Sort — O(n log n)

Modify merge sort to count inversions. When merging `left` and `right`, every time a right element is placed before a left element, it contributes to the count of the left element.

## Key Insight

Processing right-to-left + BIT converts "count smaller to the right" into "count elements already inserted that are smaller". The BIT maintains a frequency array indexed by value rank, supporting prefix sum queries in O(log n).

This pattern generalizes to:
- **#493 Reverse Pairs** — count pairs where `nums[i] > 2 * nums[j]` and `i < j`
- **#327 Count of Range Sum** — count range sums in a given range
- **#218 The Skyline Problem** — segment tree on coordinate-compressed x-values

## Edge Cases

- **All same elements**: all ranks equal, `query(rk-1)` always returns 0. All counts are 0.
- **Strictly decreasing** `[5,4,3,2,1]`: each element has count = its position from right.
- **Negative numbers**: coordinate compression handles negative values — they get positive ranks like everything else.

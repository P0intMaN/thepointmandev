## Problem Statement

Given an integer array `nums`, design a class that supports:
- `void update(int index, int val)` — update `nums[index]` to `val`.
- `int sumRange(int left, int right)` — return the sum of elements between indices `left` and `right` (inclusive).

Both operations must run in **O(log n)** time.

```java
NumArray obj = new NumArray([1, 3, 5]);
obj.sumRange(0, 2); // returns 9
obj.update(1, 2);   // nums = [1, 2, 5]
obj.sumRange(0, 2); // returns 8
```

## Approach 1: Binary Indexed Tree (BIT) — O(log n)

BIT is simpler than a full Segment Tree for sum queries with point updates:

```java
class NumArray {
    private int[] bit, nums;
    private int n;

    public NumArray(int[] nums) {
        this.n = nums.length;
        this.nums = new int[n];
        this.bit  = new int[n + 1];
        for (int i = 0; i < n; i++) update(i, nums[i]);
    }

    public void update(int index, int val) {
        int delta = val - nums[index];
        nums[index] = val;
        for (int i = index + 1; i <= n; i += i & (-i)) bit[i] += delta;
    }

    public int sumRange(int left, int right) {
        return prefix(right) - prefix(left - 1);
    }

    private int prefix(int i) {
        int sum = 0;
        for (++i; i > 0; i -= i & (-i)) sum += bit[i];
        return sum;
    }
}
```

**How BIT works**: the `i & (-i)` operation isolates the lowest set bit of `i`. This creates a tree structure where each BIT index stores the sum of a specific range — the range length equals the lowest set bit of the index.

## Approach 2: Segment Tree — O(log n)

```java
class NumArray {
    private int[] tree;
    private int n;

    public NumArray(int[] nums) {
        n = nums.length;
        tree = new int[4 * n];
        build(nums, 1, 0, n - 1);
    }

    private void build(int[] nums, int node, int start, int end) {
        if (start == end) { tree[node] = nums[start]; return; }
        int mid = (start + end) / 2;
        build(nums, 2*node, start, mid);
        build(nums, 2*node+1, mid+1, end);
        tree[node] = tree[2*node] + tree[2*node+1];
    }

    public void update(int index, int val) {
        update(1, 0, n-1, index, val);
    }

    private void update(int node, int start, int end, int idx, int val) {
        if (start == end) { tree[node] = val; return; }
        int mid = (start + end) / 2;
        if (idx <= mid) update(2*node, start, mid, idx, val);
        else            update(2*node+1, mid+1, end, idx, val);
        tree[node] = tree[2*node] + tree[2*node+1];
    }

    public int sumRange(int left, int right) {
        return query(1, 0, n-1, left, right);
    }

    private int query(int node, int start, int end, int l, int r) {
        if (r < start || end < l) return 0;
        if (l <= start && end <= r) return tree[node];
        int mid = (start + end) / 2;
        return query(2*node, start, mid, l, r)
             + query(2*node+1, mid+1, end, l, r);
    }
}
```

## Complexity

| | Build | Update | Query | Space |
|---|---|---|---|---|
| Prefix Sum (no update) | O(n) | O(n) | O(1) | O(n) |
| BIT | O(n log n) | O(log n) | O(log n) | O(n) |
| Segment Tree | O(n) | O(log n) | O(log n) | O(4n) |

## Key Insight

This problem is the entry point for Segment Tree and BIT. The comparison reveals when each is appropriate:

- **No updates**: prefix sum — build once in O(n), query in O(1)
- **Point updates + range sums**: BIT — O(log n) both, simpler code
- **Range updates + range queries**: Segment Tree with lazy propagation — BIT can't do this cleanly

The BIT's `i & (-i)` trick is the essence of its efficiency. The segment tree's recursive structure makes it more generalizable to min/max and other non-invertible operations.

## Edge Cases

- **Single element array**: tree has one leaf, BIT has one entry. Both work correctly.
- **Update same index twice**: `delta = val - nums[index]` correctly applies only the difference.
- **sumRange(i, i)**: query a single element — returns `nums[i]` after updates.

## Problem Statement

Given an `m x n` matrix `heights`, find a path from top-left `(0,0)` to bottom-right `(m-1,n-1)` that minimizes the **maximum absolute difference** between adjacent cells along the path.

```
Input:  heights = [[1,2,2],[3,8,2],[5,3,5]]
Output: 2
Explanation: path [1,3,5,3,5] has max diff 2

Input:  heights = [[1,2,3],[3,8,4],[5,3,5]]
Output: 1
```

## Approach: Dijkstra with Modified Cost — O(mn log mn)

Instead of minimizing total cost, we minimize the **maximum edge cost** along the path. Define the "distance" to a cell as the minimum possible "max effort" to reach it.

Use Dijkstra with state `(effort, row, col)` where `effort = max absolute difference` along the best path so far.

```java
public int minimumEffortPath(int[][] heights) {
    int rows = heights.length, cols = heights[0].length;
    int[][] dist = new int[rows][cols];
    for (int[] row : dist) Arrays.fill(row, Integer.MAX_VALUE);
    dist[0][0] = 0;

    // PQ: {effort, row, col}
    PriorityQueue<int[]> pq = new PriorityQueue<>((a, b) -> a[0] - b[0]);
    pq.offer(new int[]{0, 0, 0});

    int[][] dirs = {{0,1},{0,-1},{1,0},{-1,0}};

    while (!pq.isEmpty()) {
        int[] curr = pq.poll();
        int effort = curr[0], r = curr[1], c = curr[2];

        if (r == rows-1 && c == cols-1) return effort;
        if (effort > dist[r][c]) continue; // stale entry

        for (int[] d : dirs) {
            int nr = r + d[0], nc = c + d[1];
            if (nr < 0 || nr >= rows || nc < 0 || nc >= cols) continue;

            int edgeEffort = Math.abs(heights[nr][nc] - heights[r][c]);
            int newEffort = Math.max(effort, edgeEffort); // max along path

            if (newEffort < dist[nr][nc]) {
                dist[nr][nc] = newEffort;
                pq.offer(new int[]{newEffort, nr, nc});
            }
        }
    }
    return dist[rows-1][cols-1];
}
```

**Key difference from standard Dijkstra**: edge "cost" is `max(current_effort, |h[nr][nc] - h[r][c]|)` — not a sum. We're minimizing the bottleneck, not the total.

## Alternative: Binary Search + BFS — O(mn log(max_height))

Binary search on the answer `k`. For a given `k`, check if there's a path where all adjacent differences ≤ k (BFS/DFS).

```java
public int minimumEffortPath(int[][] heights) {
    int lo = 0, hi = 1_000_000;
    while (lo < hi) {
        int mid = lo + (hi - lo) / 2;
        if (canReach(heights, mid)) hi = mid;
        else lo = mid + 1;
    }
    return lo;
}

private boolean canReach(int[][] heights, int k) {
    int rows = heights.length, cols = heights[0].length;
    boolean[][] visited = new boolean[rows][cols];
    Queue<int[]> q = new LinkedList<>();
    q.offer(new int[]{0, 0});
    visited[0][0] = true;
    int[][] dirs = {{0,1},{0,-1},{1,0},{-1,0}};
    while (!q.isEmpty()) {
        int[] cell = q.poll();
        int r = cell[0], c = cell[1];
        if (r == rows-1 && c == cols-1) return true;
        for (int[] d : dirs) {
            int nr = r+d[0], nc = c+d[1];
            if (nr>=0 && nr<rows && nc>=0 && nc<cols && !visited[nr][nc]
                && Math.abs(heights[nr][nc] - heights[r][c]) <= k) {
                visited[nr][nc] = true;
                q.offer(new int[]{nr, nc});
            }
        }
    }
    return false;
}
```

## Complexity

| Approach | Time | Space |
|---|---|---|
| **Dijkstra (bottleneck)** | **O(mn log mn)** | **O(mn)** |
| Binary search + BFS | O(mn log(max)) | O(mn) |
| Union-Find | O(mn log mn) | O(mn) |

## Key Insight

Dijkstra generalizes beyond "sum of costs" — it works for any monotone path cost function. Here, the path cost is the maximum edge weight seen so far, which is monotone (adding edges can only increase or maintain the max). The greedy property holds: always expand the path with the lowest current max effort.

## Edge Cases

- **Single cell**: effort = 0
- **Uniform grid**: all differences are 0 → answer = 0
- **Must go through a mountain peak**: answer = that peak's difference

## Problem Statement

Given an integer array `nums` of **unique** elements, return all possible subsets (the power set). The solution set must not contain duplicate subsets. Return the solution in any order.

```
Input:  nums = [1,2,3]
Output: [[],[1],[2],[1,2],[3],[1,3],[2,3],[1,2,3]]

Input:  nums = [0]
Output: [[],[0]]
```

## Approach 1: Backtracking — O(n · 2ⁿ)

At each position, we have two choices: include the element or skip it and move to the next. The `start` index prevents re-using elements (no `[2,1]` if we already have `[1,2]`).

```java
public List<List<Integer>> subsets(int[] nums) {
    List<List<Integer>> result = new ArrayList<>();
    backtrack(nums, 0, new ArrayList<>(), result);
    return result;
}

private void backtrack(int[] nums, int start, List<Integer> current, List<List<Integer>> result) {
    result.add(new ArrayList<>(current)); // add current subset (including empty)

    for (int i = start; i < nums.length; i++) {
        current.add(nums[i]);                           // choose
        backtrack(nums, i + 1, current, result);        // explore
        current.remove(current.size() - 1);             // unchoose
    }
}
```

Every node in the recursion tree is a valid subset — add to result immediately (no base case needed). The `i + 1` ensures we never revisit earlier elements.

## Approach 2: Bit Manipulation — O(n · 2ⁿ)

For n elements, there are 2ⁿ subsets. Each subset corresponds to a bitmask from 0 to 2ⁿ-1 where bit j indicates whether `nums[j]` is included.

```java
public List<List<Integer>> subsets(int[] nums) {
    int n = nums.length;
    List<List<Integer>> result = new ArrayList<>();

    for (int mask = 0; mask < (1 << n); mask++) {
        List<Integer> subset = new ArrayList<>();
        for (int j = 0; j < n; j++) {
            if ((mask & (1 << j)) != 0) {
                subset.add(nums[j]);
            }
        }
        result.add(subset);
    }
    return result;
}
```

Concise and non-recursive. Works well for small n (n ≤ 20 before int overflow).

## Decision Tree

```
backtrack(start=0, current=[])
├── add []
├── i=0: current=[1]
│   ├── add [1]
│   ├── i=1: current=[1,2]
│   │   ├── add [1,2]
│   │   └── i=2: current=[1,2,3] → add [1,2,3]
│   └── i=2: current=[1,3] → add [1,3]
├── i=1: current=[2]
│   ├── add [2]
│   └── i=2: current=[2,3] → add [2,3]
└── i=2: current=[3] → add [3]
```

Total: 8 subsets = 2³ ✓

## Complexity

| | Time | Space |
|---|---|---|
| Backtracking | O(n · 2ⁿ) | O(n · 2ⁿ) |
| Bit mask | O(n · 2ⁿ) | O(n · 2ⁿ) |

2ⁿ subsets, each up to length n to copy → O(n · 2ⁿ) total. This is unavoidable — the output itself is O(n · 2ⁿ).

## Key Insight

Subsets is the foundation of backtracking. Every more complex backtracking problem adds one of:
- A **size constraint** → Combinations (#77)
- A **sum constraint** → Combination Sum (#39)
- **Ordering** → Permutations (#46)
- **Grid validity** → Word Search (#79)

Master the `start` index pattern and the add-before-loop structure here.

## Edge Cases

- **Empty array**: one subset — the empty set `[[]]`. The loop doesn't execute; initial `result.add([])` is correct.
- **Single element**: `[[],[0]]`.
- **All zeros** (with duplicates): for this problem, elements are guaranteed distinct. For duplicates, see **#90 Subsets II** — sort first, skip duplicates.
